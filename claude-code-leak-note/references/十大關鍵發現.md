# 十大關鍵發現

> 從 75 份報告、~884KB 分析內容中提煉的最重要洞察

## 1. Agent Loop 是精密的有狀態機

Claude Code 的核心不是簡單的「問答循環」，而是帶有完整狀態管理、工具執行緩衝、feedback 注入機制的有狀態執行機。`query.js` 管理的 stop reason 判斷、tool result 回注、interrupt 處理形成了生產級 LLM harness 的完整範本。

→ 詳見 [[Agent Loop 核心執行機制]]

## 2. Context Engineering 比 Prompt Engineering 更重要

系統投入大量工程資源在 context 管理：prompt cache 對齊、messages 正規化、compaction 策略。「讓模型看到什麼」遠比「如何說話給模型聽」更決定性能。

→ 詳見 [[Context Engineering 多層管道]]、[[Prompt Cache 策略與 Break Detection]]

## 3. 安全架構採用七層縱深防禦

從 bash 解析器層（tree-sitter）到 UI 確認層，實作了完整的防禦縱深。`bashSecurity.ts` 的 AST 解析路徑遠比字串匹配複雜，prompt injection 和命令注入被當作嚴肅的系統威脅處理。

→ 詳見 [[七層縱深防禦模型]]、[[Security 設計模式集]]

## 4. 記憶系統有五個獨立子系統並存

Auto Memory（持久）、Session Memory（當次）、MagicDocs（文件）、Team Memory（共享）、AutoDream（整合）構成複雜的記憶生態。每個子系統有獨立的觸發條件、存儲格式和生命週期，但共享同一套 forked agent 基礎設施。

→ 詳見 [[Memory 五大子系統架構]]

## 5. 82 個未公開 Feature Flags 揭示產品路線圖

`bun:bundle` 的 `feature()` 系統中包含大量尚未對用戶開放的功能。這些功能完整實作但以 flag 隱藏，說明產品具備遠超當前版本的技術儲備。

→ 詳見 [[82 個未公開 Feature Flags]]、[[Beta Features 與 Feature Flags 系統]]

## 6. Coordinator/Swarm 是真正的多 Agent 系統

具備完整 multi-agent 基礎設施：Coordinator Mode 的任務分解調度、Swarm 系統的 Team Lead/Teammate 協作、跨 worktree 的隔離執行。這是目前最完整的生產級 LLM multi-agent 實作之一。

→ 詳見 [[Coordinator Mode 多 Agent 協調]]、[[Swarm 與 Teammate 多 Agent 協作]]

## 7. 成本追蹤達到毫秒級精度

`cost-tracker.ts` 實現了跨 session 的精確成本追蹤、prompt cache 命中偵測、rate limit header 解析。系統能追蹤 cache break 的具體原因，成本最佳化是一等公民工程需求。

→ 詳見 [[成本追蹤架構]]、[[Cache 穩定性工程模式]]

## 8. Skills 系統實現「可程式化的 prompt 擴充」

Skills 不是簡單的文本模板，而是完整的「mini agent」框架：獨立的工具授權、可選的子 agent 隔離、防衛性的 `getPromptForCommand` 模式。

→ 詳見 [[SkillTool 與 Skills 系統]]、[[Skills vs Tools 設計哲學]]

## 9. Prompt 設計有系統化的工程模式

從 prompt 原始碼提煉出「三明治強化」、「工具偏好金字塔」、「角色綁定」等 8+ 設計模式。Prompt Engineering 已從藝術演化為工程學科。

→ 詳見 [[Prompt Engineering 設計模式集]]、[[Tool Prompt 設計模式集]]

## 10. 系統為企業/團隊場景做了深度設計

Policy Limits 的組織管控、Team Memory 的共享記憶、多 provider 支援（AWS Bedrock/GCP Vertex）等功能說明目標市場明確包含企業客戶。

→ 詳見 [[Policy Limits 團隊管控]]、[[Team Memory 跨用戶共享]]、[[模型配置與 Provider 支援]]
