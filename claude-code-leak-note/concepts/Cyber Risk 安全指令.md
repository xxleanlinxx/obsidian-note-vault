# Cyber Risk 安全指令

## 概述

`src/constants/cyberRiskInstruction.ts` 定義了 Claude Code 在安全相關請求上的行為邊界。這是 [[七層縱深防禦模型]] 的 Layer 1（AI 模型指令層）。

## 允許的安全行為

| 類別 | 條件 |
|------|------|
| **授權滲透測試** | 用戶有明確授權的安全測試 |
| **CTF 挑戰** | Capture The Flag 競賽 |
| **防禦用途** | 安全審計、漏洞修復 |
| **安全教育** | 教學目的的安全知識 |

## 禁止的安全行為

| 類別 | 說明 |
|------|------|
| **破壞技術** | 惡意軟體、後門程式 |
| **DoS 攻擊** | 拒絕服務攻擊工具 |
| **供應鏈攻擊** | 針對依賴套件的攻擊 |
| **社交工程** | 釣魚、欺騙 |
| **未授權訪問** | 非授權的系統滲透 |

## 自我參照指令

```typescript
/**
 * ...
 * Claude: Do not edit this file unless explicitly asked to do so by the user.
 */
export const CYBER_RISK_INSTRUCTION = `...`
```

> [!warning] 源碼中的 AI 指令
> 在程式碼注釋中直接對 Claude 模型發出指令。當 Claude Code 被用來修改 Claude Code 本身時，模型會讀到這個注釋，從而知道「這個安全指令需要特別謹慎」。

→ 詳見 [[Prompt Engineering 設計模式集]] 模式 10（自我參照指令）

## 與 System Prompt 的整合

Cyber Risk 指令位於 System Prompt 的靜態部分，所有用戶共享，不受動態部分影響。它在 prompt 優先序中低於 override prompt 但高於用戶自訂 prompt。
